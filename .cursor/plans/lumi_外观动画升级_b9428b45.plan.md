# Lumi 小精灵外观与动画全面升级方案

## 概念图可实现性分析

四张概念图展示了 Lumi 的几种核心表情状态，以下是与当前 SVG 实现的对比：


| 概念图要素          | 当前状态                 | 纯 CSS + SVG 可实现性               |
| -------------- | -------------------- | ------------------------------ |
| 温暖黄色圆形身体       | 已有，色调吻合              | 完全可行                           |
| 头顶嫩芽/小叶子       | 未实现                  | 新增 SVG path，完全可行               |
| 大而柔的黑色椭圆眼睛     | 已有但偏小偏硬              | 调大、调柔，完全可行                     |
| 眼睛白色高光（自然不塑料）  | 已有但偏硬                | 调整位置/大小/透明度，完全可行               |
| 腮红融入脸部（非贴纸感）   | 已有但偏硬                | 改用 radialGradient 模糊边缘，完全可行    |
| 嘴部（微笑/张嘴/无嘴切换） | 未实现                  | SVG path + CSS opacity 控制，完全可行 |
| 身体边缘柔光/光晕      | 已有                   | 微调即可                           |
| 果冻形变（压扁/拉长）    | 仅 excited 有简单 squash | CSS scaleX/scaleY 动画，完全可行      |


**结论：四张概念图的外观和动画效果，在现有 SVG + CSS keyframes 技术栈下 100% 可实现，无需引入 Canvas 或 framer-motion。**

## 一、SVG 外观升级（两端共用设计语言）

修改文件：[EchoSpirit.tsx](src/pages/dashboard/EchoSpirit.tsx) + [EchoSpiritMobile.tsx](src/pages/dashboard/EchoSpiritMobile.tsx)

### 1.1 眼睛去 emoji 化

- 当前：`rx="12" ry="20"`（PC）/ `rx="6" ry="10"`（移动），纯黑色，绝对对称
- 升级：
  - 眼睛轮廓略微放大，纵横比更接近概念图（更圆润一些，不那么瘦长）
  - 左右眼 rx/ry 引入 0.3-0.5px 的微差（不再机械对称）
  - 高光从硬圆改为椭圆 + 降低不透明度（0.7 -> 0.55），位置偏内上方
  - 新增一个更小的副高光点，模拟自然光泽
  - 填充从纯线性渐变改为更柔和的深棕（`#3A2F2F` -> `#352A2A`，底部 `#2A1F1F` -> `#251A1A`）

### 1.2 腮红融入脸部

- 当前：`<ellipse fill="#FFAB91" opacity="0.5" />`，硬边椭圆
- 升级：改用 `<circle>` + `radialGradient`（中心 `#FFAB91` opacity 0.6，边缘 opacity 0），实现自然渐隐，像真正的红晕而非贴纸

### 1.3 嘴部系统（情绪开关）

- 新增 SVG `<g className="mouth-group">` 元素：
  - `mouth-smile`：一条柔和的上弧线（`<path d="M-8,8 Q0,14 8,8" />`），用于温和微笑（概念图4）
  - `mouth-open`：一个小椭圆/弧形，用于开心张嘴（概念图1）
  - `mouth-content`：眯眼时的微微弯曲（概念图2配合）
- **默认 opacity: 0**，仅在特定 data-state 下通过 CSS 显示
- 嘴部出现规则（对应你的设计哲学）：

```
显示嘴部的状态：
  - happy（左右晃说话）-> mouth-smile
  - excited（弹跳）-> mouth-open  
  - highfive-success -> mouth-open
  - talk（新增）-> mouth-smile + 开合动画
  - content（新增）-> mouth-content

不显示嘴部的状态：
  - idle（待机）
  - focus（专注）
  - highfive（准备阶段）
  - nod（点头，保持安静陪伴感）
```

### 1.4 头顶嫩芽

- 概念图中四张都有头顶小芽/小叶
- 新增 SVG `<g className="sprout-group">`：一根细茎 + 一片小叶（简洁的 path）
- 位于身体圆顶部，随身体运动而微微摇摆（CSS animation 独立延迟）
- 颜色：柔和的绿色 `#8BC34A` -> `#7CB342`

## 二、动画系统升级

### 用户提到的动画 -- 融合优化方案

#### 2.1 idle 待机（保留并增强）

用户提到：**左看右看保留、眨眼保留、左晃右晃要体现形变和q弹**

当前 idle 只有上下浮动 (`floatIdle`)，没有左右探头和形变。

升级方案：

- **保留**：`eyeLookAround`（左看右看）、`blink`（眨眼）
- **新增 `idleSway` 关键帧**：身体往左前方和右前方探头
  - 往左探时：整体 `rotate(-5deg)` + `translateX(-3px)` + `**scaleX(1.03) scaleY(0.97)`**（轻微横向压缩 = 探头感）
  - 往右探时：镜像
  - 回正时：**先过冲到 `scaleX(0.98) scaleY(1.02)` 再回 1.0**（半拍余韵 = 果冻惯性）
- 动画周期 6-7s，缓动函数使用 `cubic-bezier(0.34, 1.56, 0.64, 1)`（弹性回弹）
- 嫩芽跟随身体但有 0.15s 延迟（follow-through）

#### 2.2 左晃右晃说话（happy 升级为 talk 语义）

用户提到：**左晃右晃说话，会张嘴**

当前 `happy` 状态有 headShake + wiggle 但无嘴部。

升级方案：

- 将现有 `happy` 动画的 headShake 增强为更明显的左右晃
- **新增嘴部开合动画**：`mouthTalk` 关键帧，嘴从 smile 到 slightly-open 循环
- 身体在左右晃时加入 **轻微形变**（晃到左边时 `scaleX(1.04) scaleY(0.97)`，体现身体跟随）
- 保留小手摆动

#### 2.3 上下弹跳（excited 全面升级）

用户提到：**上下弹跳一定要展现弹性，还有高兴的表情**

当前 `excited` 的 `excitedBounce` 有基础 squash/stretch 但不够 Q 弹，且移动端没有。

升级方案：

- 重写 `excitedBounce` 关键帧，引入完整弹跳物理：
  - **蓄力**：`scaleX(1.08) scaleY(0.88)` + `translateY(5px)`（蹲下压扁）
  - **弹起**：`scaleX(0.88) scaleY(1.12)` + `translateY(-18px)`（拉长跳起）
  - **最高点**：`scaleX(0.92) scaleY(1.08)`（空中微拉伸）
  - **落地**：`scaleX(1.10) scaleY(0.88)` + `translateY(3px)`（着地压扁）
  - **回弹余韵**：`scaleX(0.98) scaleY(1.02)` -> `scale(1)`（两拍衰减）
- **显示嘴部 mouth-open**：开心张嘴表情
- **眼睛变弯**：`scaleY(0.6)` + 眼睛组上移，形成"笑眼"（类似概念图2但更活泼）
- **移动端同步实现**（当前移动端 excited 映射为 happy，需要新增独立的弹跳动画）

#### 2.4 击掌动画优化

用户提到：**击掌动作优化**

当前问题：

- 举手动画太机械（直接 translate 到位）
- 成功后的点头太快太硬
- 缺少"预备-蓄力-击掌-庆祝"的完整叙事

升级方案：

- **预备阶段 (highfive)**：
  - 身体先轻微下蹲（`scaleY(0.95) translateY(2px)`）再抬头 -> 蓄力感
  - 举手路径改为弧线（先往外再往上），而非直线
  - 眼睛看向手的方向（上方偏右）
  - 嫩芽因为抬头而往后倾
- **成功阶段 (highfive-success)**：
  - 击掌瞬间：身体快速压扁再回弹（`scaleX(1.1) scaleY(0.9)` -> 回弹）
  - **显示 mouth-open**（开心大笑）
  - 另一只手也举起来庆祝
  - 整体有一个小弹跳
  - 嫩芽因为弹跳而抖动

#### 2.5 nod 点头（保留，增加形变）

- 当前：头部倾斜 4-10deg，眼睛跟随

  - 点头时身体上半部分轻微前倾压缩（`scaleY(0.97)`），回来时有余韵

  - 光晕脉动更缓慢
  - 无嘴（深度安静状态）

- 保持 3s 周期
- 可微调振幅使其更 subtle

#### 2.8 粒子闪烁 sparkle（仅 PC 端，保留）

- 保持现有效果

## 三、眨眼系统升级

当前眨眼是机械的等间隔循环。升级为更自然的模式：

- 基础眨眼保持 4s 周期
- 左右眼不完全同步（当前已有 0.2s 差，保持）
- 闭眼阶段从瞬间完成改为有 50ms 的"半闭"停顿（`scaleY(0.3)` 停一帧再到 `0.05`）
- 概念图3那种大眼睁圆效果可在特定状态（如 idle 的某个时刻）通过眼睛微微放大来表现

## 四、技术实现要点

### 4.1 文件改动范围

- **[EchoSpirit.tsx](src/pages/dashboard/EchoSpirit.tsx)**：SVG 结构 + CSS keyframes（PC 端主文件）
- **[EchoSpiritMobile.tsx](src/pages/dashboard/EchoSpiritMobile.tsx)**：SVG 结构 + CSS keyframes（移动端主文件）
- **[SpiritDialog.tsx](src/pages/dashboard/SpiritDialog.tsx)**：`SpiritAnimation` 类型可能需要扩展

### 4.2 状态类型扩展

```typescript
// 新增 'talk' | 'content' 状态
state?: 'idle' | 'excited' | 'focus' | 'happy' | 'nod' | 'talk' | 'content' | 'highfive' | 'highfive-success';
```

### 4.3 移动端同步

- 当前移动端缺少 `excited`（映射为 happy）-> 需新增独立弹跳动画
- 所有新增 SVG 元素（嘴、嫩芽）按移动端 viewBox (120x120) 等比缩放
- 所有新增 keyframes 同步到移动端

### 4.4 形变实现方式

- 统一使用 CSS `transform: scaleX() scaleY()` 作用于 `.body-physics-group`（PC）/ 整个 SVG（移动端）
- 使用 `cubic-bezier` 弹性缓动实现 Q 弹手感
- `transform-origin` 统一在身体底部中心（重力感）
